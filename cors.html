CORS Misconfiguration

CORS controls which external origins can access a web application's responses.

What is CORS?

CORS (Cross-Origin Resource Sharing) controls which external origins can access a web application's responses. A misconfiguration can let an attacker read sensitive data using a victim's authenticated session.

CORS Testing Tools

Corser (Best Overall - Fast & Accurate)

corser -i urls.txt -o corser_results.txt

Corsy (Best Secondary - Payload-Based)

python3 corsy.py -i urls.txt -t 20

Manual CORS Testing

Basic Origin Test

curl -I -H "Origin: https://evil.com" https://target.com

Null Origin Test

curl -I -H "Origin: null" https://target.com

Subdomain Origin Test

curl -I -H "Origin: https://subdomain.target.com" https://target.com

Vulnerable Headers

  • Access-Control-Allow-Origin: *
  • Access-Control-Allow-Origin: null
  • Access-Control-Allow-Origin: https://evil.com
  • Access-Control-Allow-Credentials: true (with wildcard)
